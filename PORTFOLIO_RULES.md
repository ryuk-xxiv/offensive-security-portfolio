# Portfolio Publishing Rules

Before publishing any content, confirm that it does not include:

- Client names or identifying details
- Real IP addresses, domains, hostnames, usernames, or emails
- Credentials, hashes, keys, tokens, or secrets
- Screenshots containing sensitive data
- Exploit instructions against real systems
- Proprietary reports, policies, or internal documentation
- Unapproved third-party lab spoilers

Use sanitized diagrams, mock data, and generalized methodology instead.
